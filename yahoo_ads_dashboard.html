プロキシサーバー セットアップマニュアル
Yahoo!広告 API × Cloud Run 連携の完全手順
目次
① 全体の流れ ② 前提条件・必要なもの ③ Google Cloud SDK のインストール ④ プロキシサーバーのデプロイ（Cloud Run） ⑤ ダッシュボードへの接続設定 ⑥ Yahoo!広告 APIの認証情報取得 ⑦ エラーと対処法
1全体の流れ
このシステムは3つのパーツで構成されています。
┌─────────────────────────────────────────────┐ ダッシュボード（このHTMLファイル） ブラウザ上で動作。認証情報を入力してデータ取得ボタンを押す │ POST /yahoo-ads/report ▼ プロキシサーバー（Cloud Run） Googleのクラウドで動作。ブラウザの代わりにYahoo!APIを呼び出す │ Yahoo!広告 API リクエスト ▼ Yahoo!広告 API レポートデータを返す → ダッシュボードに自動反映 └─────────────────────────────────────────────┘
2前提条件・必要なもの
必要なもの状態の確認方法
Google Cloud アカウントconsole.cloud.google.com にログインできる
Google Cloud プロジェクトプロジェクトIDを確認済み（例: composed-augury-xxxxxx）
Google Cloud SDK（gcloud）ターミナルで gcloud version が動く
yahoo-ads-cloudrun.zipClaudeからダウンロード済み
Yahoo!広告 APIアカウント申請後に発行（後でOK）
注意：Cloud Run デプロイには Google Cloud の課金が必要です。ただし低負荷な利用では月額ほぼ無料（無料枠内）に収まります。
3Google Cloud SDK のインストール
macOS (Homebrew) の場合：
# Homebrewでインストール brew install --cask google-cloud-sdk # ターミナルを再起動後、確認 gcloud version # → Google Cloud SDK 4xx.x.x と表示されればOK
インストール後、Googleアカウントでログインします：
# ブラウザが開いてGoogleログイン画面が表示される gcloud auth login # プロジェクトIDを設定 gcloud config set project YOUR_PROJECT_ID # 確認 gcloud config list
4プロキシサーバーのデプロイ（Cloud Run）
ダウンロードしたZIPを展開して、デプロイスクリプトを実行するだけです。
# ① ZIPを展開（ダウンロードフォルダに保存した場合） cd ~/Downloads unzip yahoo-ads-cloudrun.zip cd yahoo-ads-cloudrun # ② ワンコマンドデプロイ（初回は3〜5分かかります） bash deploy.sh
デプロイが完了すると以下のようなURLが表示されます：
デプロイ完了 サービスURL: https://yahoo-ads-proxy-xxxx-an.a.run.app 正常に起動しています (HTTP 200) ================================================ ダッシュボードのプロキシURLに以下を設定: https://yahoo-ads-proxy-xxxx-an.a.run.app ================================================
このURLをコピーして次のステップで使います。
2回目以降の更新も同じコマンドで自動的に上書きデプロイされます。
5ダッシュボードへの接続設定
デプロイ完了後、このダッシュボードの「Yahoo!広告 API 自動連携」パネルに情報を入力します。
入力項目入力する値
プロキシサーバーURLデプロイで表示された https://yahoo-ads-proxy-xxxx-an.a.run.app
クライアントIDYahoo!広告 APIポータルで発行されたID
クライアントシークレット同ポータルで発行されたシークレット
アクセストークンOAuth認証後に発行されるトークン
広告アカウントIDYahoo!広告管理画面の数字のID
取得開始日〜終了日レポートを取得したい期間
「設定を保存」ボタンを押すと、次回ブラウザを開いたときも入力内容が保持されます（シークレット・トークンを除く）。
入力後、「データを取得」ボタンが青くなったらクリックするとデータが自動取得されます。
6Yahoo!広告 APIの認証情報取得
Step 1：Yahoo!広告 APIポータルでデベロッパー登録
https://ads-developers.yahoo.co.jp/ja/ads-api/startup-guide/apply-api-use.html
Step 2：審査通過後、クライアントID・シークレットが発行される（数日〜1週間）
Step 3：OAuth認証フローでアクセストークンを取得
# 認可URLにブラウザでアクセス（CLIENT_IDを自分のIDに変更） https://biz-oauth.yahoo.co.jp/oauth/v1/authorize?response_type=code&client_id=CLIENT_ID&redirect_uri=https://localhost&scope=yahooads # ログイン→承認後にリダイレクトURLのcodeパラメータを取得 # 例: https://localhost?code=XXXXXXXXXX # プロキシの /token エンドポイントでトークン交換 curl -X POST https://yahoo-ads-proxy-xxxx-an.a.run.app/token \ -H "Content-Type: application/json" \ -d '{ "clientId": "あなたのクライアントID", "clientSecret": "あなたのシークレット", "code": "取得したcode", "redirectUri": "https://localhost" }' # → access_token が返ってくるのでダッシュボードに入力
7エラーと対処法
エラー原因対処
gcloud: command not foundSDK未インストール③の手順でインストール後ターミナル再起動
One or more users... organization policy組織ポリシーでCloud Functions禁止Cloud Runを使う（このマニュアルの手順）
HTTP 401 UNAUTHORIZEDアクセストークン切れ/refresh エンドポイントでトークン更新
HTTP 504 TIMEOUTレポート生成が90秒超取得期間を短くして再試行
データが0件期間内にデータなし広告アカウントIDと取得期間を確認
トラブル時はターミナルで以下を実行するとCloud Runのログが確認できます：
gcloud run services logs read yahoo-ads-proxy --region=asia-northeast1 --limit=50
